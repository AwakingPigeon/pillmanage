## 目标
- 删除页面中的“专为抑郁症患者设计”文案（保持中性表述）。
- 增加预设药物清单：先选分类，再选具体药物；支持填写“其他药物名”。
- 完成本地提交与远程推送到 `origin/main`。

## 影响范围
- Web 单页：`index.html`（头部文案、设置页的药物输入交互）。
- RN 应用：`App.tsx`（药物设置页增加预设分类/药物选择与自定义输入）。
- 类型与存储：`DepressionMed/src/types.ts`（可选字段以保存分类），`DepressionMed/src/services/storage.ts`（兼容新增字段）。

## 实现方案（Web）
1. 删除头部文案：移除 `index.html:314` 的 `<p>专为抑郁症患者设计</p>`。
2. 预设清单数据：在 `index.html` 脚本中新增常量 `PRESET_MEDICINES`，按 `docs/medicine.md` 的分类与药物填充（如“抗抑郁药”、“抗精神病药”等）。
3. 设置页 UI：
- 在 `renderMedicationsSetup` 中为每个药物卡添加分类 `<select>` 与药物 `<select>`；药物下拉包含“其他”，当选择“其他”时显示文本输入以填写自定义药物名。
- 保存逻辑优先采用选择的药物名；若选“其他”，使用文本框值；继续沿用现有 `medications` 结构（新增 `category` 与 `useCustom`/`customName` 字段）。
4. 校验与存储：更新校验，确保最终药物名非空；存储仍用 `localStorage`，保持兼容已存数据（缺省时按旧字段回落）。

## 实现方案（RN）
1. 预设清单：在 `App.tsx` 内新增 `PRESET_MEDICINES` 常量（与 Web 相同分类），避免引入新依赖。
2. 设置页交互：
- 在“药物设置”区域增加分类选择（按钮/标签样式）与药物选择（列表按钮）。
- 提供“其他药物名”输入框；保存时将 `medicationName` 设为所选或自填；新增可选字段 `medicationCategory`。
3. 数据模型：
- 扩展 `DepressionMed/src/types.ts` 的 `MedicationConfig`，新增可选 `medicationCategory?: string`；无需迁移脚本，存储服务原样 JSON 存取兼容。

## 验证
- Web：
- 打开 `index.html`，检查头部无该文案；在设置页选择分类与药物、选择“其他”并填写；保存后在主页与记录页显示所选药物名。
- RN：
- `npm install` 后运行 `npm run web/android/ios`；在“药物”页体验分类与药物选择；保存后首页显示对应药物名，通知不受影响。
