# 抑郁症服药管理应用 - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 产品名称
抑郁服药助手 (DepressionMed)

### 1.2 产品愿景
专门为抑郁症患者解决服药健忘问题，通过简单可靠的智能提醒系统，帮助患者按时服药、坚持治疗、改善生活质量。

### 1.3 目标用户画像（抑郁症患者群体）

#### 1.3.1 核心用户群体
**抑郁症患者（100%聚焦）**
- **疾病特征**：抑郁症伴随的认知功能障碍、记忆力减退、注意力不集中
- **核心问题**：服药健忘是抑郁症治疗依从性差的主要原因
- **用户画像**：
  - 年龄：20-60岁，主要集中在中青年群体
  - 教育水平：中等以上，能够使用智能手机
  - 治疗阶段：正在接受抗抑郁药物治疗
  - 技术接受度：中等，需要简单易用的解决方案

#### 1.3.2 抑郁症患者健忘问题分析
- **记忆障碍**：短期记忆受损，容易忘记服药记录，从而导致漏服甚至多服。
- **执行功能下降**：计划和组织能力减弱，难以坚持复杂服药计划
- **动机缺乏**：抑郁导致的动力不足，影响服药主动性
- **注意力分散**：难以集中注意力完成服药确认

#### 1.3.3 核心需求优先级
1. **简单可靠的服药提醒**（P0）
2. **极简的服药确认流程**（P0）  
3. **温和鼓励的交互体验**（P1）
4. **基本的服药记录功能**（P1）
5. **隐私保护**（P1）

### 1.4 核心价值主张（聚焦抑郁症健忘问题）
- **解决健忘痛点**：专门针对抑郁症患者的记忆障碍，提供可靠的服药提醒
- **极简操作设计**：认知负荷最小的界面，让患者在注意力分散时也能轻松使用
- **温和鼓励体验**：使用支持性语言和积极反馈，减轻服药的心理负担
- **可靠提醒系统**：多重提醒机制确保不会错过任何一次服药
- **隐私安全保障**：本地数据存储，隐私信息

## 2. 核心功能需求（聚焦抑郁症健忘问题）

### 2.1 极简服药计划（P0 - MVP核心）

#### 2.1.1 基础药物设置
- **药物添加**：只需输入药物名称
- **片剂计量**：提供直观的片剂剂量选择（半片/一片/一片半/两片）
- **服药时间**：设置1-3个固定服药时间点
- **极简信息**：避免复杂的药物信息录入

#### 2.1.2 智能计划管理
- **自动重复**：每日自动重复服药计划
- **简单调整**：支持临时跳过或调整服药时间
- **进度显示**：直观的今日服药完成状态

### 2.2 可靠提醒系统（P0 - MVP核心）

#### 2.2.1 多重提醒机制
- **主提醒**：预设服药时间的准时提醒
- **延迟提醒**：15分钟后未确认的二次提醒
- **最终提醒**：30分钟后未确认的最终提醒
- **温和提醒**：使用支持性语言，避免压力

#### 2.2.2 滑动确认设计
- **滑动确认**：采用滑动操作替代点击，防止误触
- **触觉反馈**：滑动确认后提供轻微的触觉反馈
- **动画效果**：平滑的滑动动画增强操作体验
- **鼓励信息**：确认后显示积极的鼓励话语

### 2.3 基本记录功能（P1 - 重要功能）

#### 2.3.1 服药历史
- **简单记录**：自动记录每次服药时间和状态
- **月度统计**：显示本月服药完成率
- **趋势查看**：简单的服药依从性趋势图

#### 2.3.2 隐私保护
- **本地存储**：所有数据仅存储在本地设备
- **无云端同步**：避免隐私泄露风险
- **数据加密**：基础的数据加密保护

### 2.3 情绪支持功能（P1 - 扩展功能）

#### 2.3.1 温和提醒设计
- **支持性语言**：使用鼓励而非责备的提醒文案
- **积极反馈**：每次服药确认后显示随机的鼓励话语
- **状态提示**：错过服药时提供安慰性提示而非责备

#### 2.3.2 简单情绪记录
- **情绪快速标记**：服药时可快速标记当前情绪状态
- **情绪趋势**：简单的情绪变化趋势显示
- **无压力记录**：完全可选，不影响主要功能

### 2.4 温馨UI设计（P0 - 基础要求）

#### 2.4.1 色彩系统
- **主色调**：柔和的橄榄绿 (#7C9885)，传达平静和希望
- **背景色**：温暖的米白色 (#FAF8F5)，提供舒适视觉体验
- **文字颜色**：深灰色 (#333333) 确保良好可读性
- **状态颜色**：温和的绿色/橙色提示系统

#### 2.4.2 界面风格
- **圆角设计**：所有卡片和按钮采用圆角设计，减少尖锐感
- **柔和阴影**：轻微阴影提升界面层次感
- **大字体显示**：清晰易读的大号字体
- **简洁布局**：减少视觉干扰，突出核心功能

### 2.5 隐私保护功能（P0 - 基础要求）

#### 2.5.1 数据安全
- **本地存储**：所有数据仅存储在用户设备本地
- **基础加密**：简单的数据加密保护隐私
- **无云端同步**：避免任何隐私泄露风险

#### 2.4.2 用户控制
- **数据删除**：用户可随时清除所有个人数据
- **简单设置**：隐私设置简单明了，无需复杂配置

## 3. 用户界面和用户体验（抑郁症患者优化）

### 3.1 设计原则
- **极简主义**：界面极其简洁，避免信息过载
- **大字体大按钮**：便于注意力不集中时操作
- **温和配色**：使用平静的蓝色、绿色等舒缓色彩
- **一致性**：所有界面保持相同的操作逻辑

### 3.2 主要界面设计

#### 3.2.1 今日首页
- **当前服药状态**：清晰显示今日待服药物
- **大确认按钮**：超大按钮便于快速确认服药
- **简单进度**：直观的今日服药完成进度

#### 3.2.2 计划设置页
- **极简设置**：只需3步完成药物设置
- **预设时间**：提供早餐、午餐、晚餐、睡前等预设选项
- **一键保存**：设置完成后立即生效

#### 3.2.3 历史记录页
- **月度统计**：简单的本月服药完成率
- **趋势图表**：直观的服药依从性趋势
- **无复杂分析**：避免给用户带来分析压力

### 3.3 交互优化
- **一键操作**：所有主要功能都支持一键完成
- **语音支持**：可选语音确认服药功能
- **震动反馈**：操作成功后的温和震动

## 4. 技术要求（抑郁症服药助手）

### 4.1 技术栈选择
- **前端框架**：React Native（跨平台支持）
- **本地存储**：SQLite本地数据库
- **平台支持**：iOS 13+/Android 9.0+
- **提醒功能**：原生推送通知支持

### 4.2 核心功能要求
- **离线工作**：所有核心功能支持完全离线使用
- **本地存储**：所有数据仅存储在用户设备本地
- **提醒可靠性**：确保服药提醒准时可靠
- **简单稳定**：应用运行稳定，不占用过多资源

### 4.3 性能要求
- **快速启动**：应用启动时间不超过2秒
- **低内存**：内存占用控制在50MB以内
- **省电优化**：后台提醒服务优化电池使用

## 8. 公益项目原则声明

### 8.1 项目定位
- **纯公益性质**：本项目为完全公益性质，不涉及任何商业盈利
- **用户免费使用**：所有功能对用户完全免费开放
- **无商业合作**：不引入任何商业机构合作（药店、保险公司等）
- **无数据商业化**：用户数据仅用于改善产品体验，绝不用于商业用途

### 8.2 功能边界
- **专注核心功能**：仅提供基础的服药提醒和记录功能

- **本地数据存储**：所有数据仅存储在用户设备本地
- **无第三方集成**：不集成任何第三方服务或API

### 8.3 运营原则
- **开源透明**：项目代码开源，运营过程透明
- **社区驱动**：功能开发基于用户真实需求
